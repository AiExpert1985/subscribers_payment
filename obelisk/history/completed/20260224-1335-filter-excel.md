
# Task: Filter Excel Import by Account Number Starting with 10

## Goal
To ensure that only relevant payment records are imported from Excel files by discarding any rows where the account number does not begin with "10".

## Scope
✓ Included: Modifying the Excel import logic to check the `account_number` column.
✓ Included: Trimming leading/trailing whitespace from the account number before checking.
✓ Included: Silently skipping rows that do not meet the criteria (no error messages or UI interruptions).
✓ Included: Ensuring the skipped rows do not cause the entire import file to be marked as failed if at least one valid row exists.
✗ Excluded: Modifying how existing payments or accounts are handled.
✗ Excluded: Showing a summary of skipped rows in the UI.

## Constraints
- **Contract: Payment Immutability** — This change only affects new incoming data during import, not existing payments.
- **Contract: Import Auto-Create** — Only applied to rows that pass the filter (starting with "10").

## Execution Strategy
The import logic iterates through Excel rows and maps columns to data fields. We will inject a validation step for the resolved `account_number`. First, the value will be converted to a string and trimmed of spaces. If it does not start with "10", the row will be immediately skipped and processing will continue to the next row without throwing an error or incrementing failure counts that might halt the import.

Rows failing this check are silently skipped (return null). This exactly matches the execution strategy defined in the task.
